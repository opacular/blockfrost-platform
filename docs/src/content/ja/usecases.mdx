# ユースケース

Blockfrost プラットフォームはどのような方に向いているのでしょうか？

## Icebreaker フリートモード

**SPO とノードオペレーター向け**

Cardano ノードを Blockfrost の分散型 API レイヤーを通じて収益化する、厳選されたインフラプロバイダーのネットワークに参加できます。次のような方に最適です。

- 既存のリレーノードインフラストラクチャを活用したい SPO
- 24 時間 365 日の安定稼働が可能なノードオペレーター
- 分散型データレイヤーの形成に関心があるアーリーアダプター
- リソース使用率に変動がある組織

### 仕組み

- 段階的ロールアウト (Icebreakers) の一環として Blockfrost クラスタに参加します
- 既存のノードインフラストラクチャを使って API リクエストを処理します
- ネットワーク容量への貢献度に応じて報酬を獲得します
- 利用可能なリソースに合わせて参加規模を拡大・縮小できます

### メリット

- 既存のインフラストラクチャから新しい収益源を創出
- 数百万件のリクエストを処理してきた実績ある設計を活用
- 分散型データアクセスレイヤーの構築に参加
- 専門的なモニタリングとシームレスなアップデート

## ソリタリーモード

**開発者および企業向け**

Blockfrost プラットフォームインスタンスを独立してデプロイし、インフラストラクチャを完全に制御できます。次のような場合に最適です。

- ローカル開発環境を必要とする DApp 開発者
- 規制遵守要件のある金融機関
- 機密データを扱う組織
- 特殊なインフラ要件を持つチーム

### 仕組み

- ローカル PC からエンタープライズデータセンターまで、どこにでもデプロイ可能
- エンドポイントとデータフローを完全にコントロール
- Blockfrost プラットフォームネットワークから独立して運用

### メリット

- インフラとデータを完全に制御
- 開発から本番までのシームレスなワークフロー
- 規制対象事業者向けのコンプライアンス簡素化
- エンタープライズグレードのスケーラビリティと信頼性
- 本番対応の API 統合により、ミドルウェアの開発が不要
